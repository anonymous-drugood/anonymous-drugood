Ideally, to measure the upper-bound OOD performance, we should use the Test-To-Test strategy. [c.f. WILDS] i.e., hold the test set Test_ood fixed and train on a separate but identically-distributed training set Train_ood. However, such a strategy requires Train_ood to match the size of Train_iid, this is not feasible in our dataset, as Train_iid usually includes most of the available data. Following WILDS, we conducted the exp under "Mixed-To-Test". i.e., train a model on a mixture of data from Train_iid and Test_ood (subset of 20 percent) and then evaluate it only on the rest data of Test_ood, to provide an underrated upper bound on OOD domain performance. The results show that adding a small amount of OOD domain improves model performance in OOD Test set. This is an underestimated upper limit performance, but it indicates that method performance isn't attributable to learning difficulty, which we'll discuss in the latest version. In details, we used the default GIN and Bert to encode the small molecule and protein data. Note that the models in the table are validated on a the patical Test_ood set (subset of 80 percent), unlike the full OOD dataset used in the maintext.
